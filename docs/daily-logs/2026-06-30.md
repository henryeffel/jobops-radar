## 4. What I Learned

- A SQLAlchemy engine manages connectivity, while a session represents a unit of
  database work.
- Alembic discovers tables through `Base.metadata` and applies versioned schema
  changes instead of modifying tables manually.
- A pre-insert duplicate lookup improves the normal error path, but only a
  database unique constraint protects the invariant under concurrent requests.
- `(source, external_id)` is the correct external posting identity because the
  same external ID may exist under different providers.
- FastAPI dependency overrides allow route tests to use an isolated test
  database without touching local `jobops.db`.
- Idempotent create behavior lets a duplicate POST return the existing resource
  instead of creating another row.
- Pytest may delete and recreate a directory supplied through `--basetemp`;
  protected or locked Windows paths can therefore fail with `WinError 5`.
